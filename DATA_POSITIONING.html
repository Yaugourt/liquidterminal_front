01 · Le piege

"Avoir la meme data" est un ticket d'entree, pas un avantage. La data est deja chez HypeDexer, Hyperliquid public, HypurrScan. Un onglet se copie en un apres-midi.

Flowscan affiche deja HIP-3 + HIP-4 + Builder Intelligence. hl.eco a Wall Street + market share + yield + projects. HypurrScan a l'EVM + forensics. Si "house of all" veut dire "j'ai tous les memes onglets", tu finis 4e terminal identique mais moins connu.

La differenciation vient de ce qu'on FAIT de la data, pas de la data elle-meme.

03 · Differenciation defendable
Axe A

Donnees connectees, pas siloten.

Tous servent des dashboards agreges OU un explorer d'adresse. Personne ne connecte les objets entre eux. La colonne vertebrale de LT n'est pas une liste d'onglets, c'est un graphe d'entites.

DEX HIP-3→ Deployer→ Oracle health→ Top traders→ Vaults→ Wallets EVM→ Positions

Un seul flux, sans changer d'outil. Les dashboards deviennent des portes d'entree dans le graphe, pas le produit.

EuxDashboards agreges (onglet par onglet) OU explorer (une adresse). Les objets ne se parlent pas.
Liquid TerminalLa relation entre les donnees EST le produit. "Nansen pour HL, mais ouvert et cross-layer".

Axe B

Substrat ouvert, pas jardin ferme.

Les trois sont fermes par necessite economique. LT est MIT + API gratuite : il peut etre ce qu'aucun d'eux ne peut se permettre.

Fermes (ne peuvent pas ouvrir)
hl.eco monetise le referral. Le lock-in, c'est le trafic.
Flowscan est le lead-gen d'un SaaS B2B. Ouvrir tuerait le funnel.
HypurrScan vend son API. Ouvrir tuerait le produit.
Liquid Terminal (ouvert)
Le substrat data ouvert sur lequel le reste de l'ecosysteme construit.
API libre + widgets embeddables + data forkable (MIT).
"House of all" pris au mot : la maison ou les apps des autres habitent. Incopiable sans se saborder.
04 · Reconciliation avec le STRATEGY.md

Pas une contradiction, une precision.

"Le terminal est une commodite" = vrai pour le terminal-tas-d'onglets, faux pour le terminal-substrat-connecte-et-ouvert. La differenciation est architecturale.

SubstratLa data, connectee (graphe d'entites) et ouverte (API/MIT). Le socle differenciant.
↑ rend lisible
TerminalLa surface dense qui rend le substrat lisible, entity-first. Design System V4.
