Liquid Terminal · Strategy Brief

Le bot d'abord, le terminal ensuite.

Repositionnement fondateur d'un bien public open source sur Hyperliquid. Le terminal web est une commodite deja couverte. Le seul actif qui retient, distribue et encaisse, c'est le bot Telegram.

Open source · MIT Wedge : alertes retail HIP-3 Freemium leger Grant-funded MAJ 01/07/2026
01 · Le verdict

Liquid Terminal n'est pas un SaaS deguise. C'est un bien public OSS finance par grants et dons, dont le seul actif a retention, distribution et surface de paiement est le bot Telegram. Direction : arreter le framing terminal-first, faire du bot d'alertes retail le produit, et du site le canal d'acquisition gratuit qui le nourrit.

Verite dure 1 · Dependance

Tout le wedge repose sur l'API d'un tiers (HypeDexer). Flowscan prouve le risque : c'est deja le dashboard gratuit d'une boite data B2B (Hydromancer). LT risque d'etre ce dashboard pour HypeDexer. Le moat se batit sur la distribution retail, pas sur la data brute.

Verite dure 2 · Echelle

2 a 4k$/mois de bot n'est pas un business, c'est un tip jar qui ne paie pas un dev. Donc grants et dons en priorite, revenu bot en complement recurrent. On ne se ment pas sur le TAM retail HL, encore inconnu.

02 · Le wedge (la chose UNE)

Le bot d'alertes Hyperliquid que le retail garde ouvert 24/7. Les autres affichent la data (HIP-3 compris) dans des dashboards. Aucun ne la pousse en alerte au retail sur Telegram. LT possede cette couche de livraison.

L'exclusivite n'est pas la data. C'est le push. Le site nourrit le bot, pas l'inverse.

04 · Etat honnete du produit

Forces reelles

+Bot Telegram livre (grammY v1.4.1) : Fill + Liquidation alerts, wallet tracking, filtres. Seul actif a retention et paiement.
+Market + Explorer matures sur data reelle (spot/perp/HIP-3, address/block/tx/validator/vaults).
+Dashboard V4 : 8 sections cablees (Network Pulse, Protocol Revenue 5 sources, Live Activity).
+Archi back propre : 4 couches, circuit breaker, rate limiter, Redis, 4 DB isolees.
+Design System V4 discipline, "real data only, no fake sparkline". Rare.

Dettes reelles (par gravite)

!Instabilite backP0 pollers/WS saturent les pools DB, crashs confirmes. Un terminal qui tombe n'a aucune credibilite.
!Dependance dure HypeDexer single point of failure externe ET concurrent potentiel.
!72 commits V4 non mergesP0 prod n'est pas le code lu. Rien de neuf ne compte tant que ce retard existe.
!Fragmentation : mockups a fausses donnees, 8 modeles Telegram parkes, "coming soon" eparpilles.
!Zero monetisation implementee : aucun quota, aucun tier, aucun paiement.
